## 1. Navbar 모바일 로그인 버튼 노출 (components/Navbar.tsx)
현재 모바일(md:hidden)은 햄버거 안에만 로그인(auth.signin)이 있음(라인 168). → **모바일 헤더에 로그인 버튼을 햄버거 밖으로 직접 노출.**
- 비로그인: 헤더 우측에 "로그인" 버튼(brand 배경) + 햄버거. 로그인 상태: 계정 아바타/이름 + 햄버거.
- 근거: 5크레딧이 가입 보너스 → 가입 유도가 핵심 전환인데 로그인이 햄버거에 묻히면 손해.
- 터치 44px+, 토큰만.

## 2. 랜딩 개편 (app/page.tsx + messages/ko.ts·en.ts)

### 2a. 히어로 통계 (3개)
- 기존 "12종 / 50종+ / 10초" → **"30종+(불량 유형) / 50종+(수지 지원) / 범용~슈퍼EP(커버 범위)"**
- 12종이 제품을 좁혀 보이게 함 → 30종+로. 3번째를 속도 대신 커버 범위로(범위 강조가 beachhead 정합).

### 2b. 5크레딧 무료 배너 (히어로 CTA 아래 신규)
- CTA("무료로 추정해보기") 바로 아래: **"가입하면 5크레딧 무료 · 5번 무료 진단"** (gift 아이콘, brand-tint 배경).
- 핵심 혜택(가입 5크레딧 one-time)을 첫 화면에 노출 → 가입 유도.

### 2c. 무료 카피 정리 (오해 제거)
- `landing.cta_sub` "회원가입 없이 바로 사용 가능합니다" → **삭제 또는 "샘플은 가입 없이 체험, 진단은 가입 후 5크레딧 무료"**로 정정. (실제론 샘플 1개만 비로그인, 진짜 진단은 로그인+크레딧 → 오해 소지 제거)
- `pricing.*`는 이미 "가입 즉시 5크레딧 무료"라 정확 — 유지.
- `chat.limit_msg`의 폐기된 "무료 플랜" 표현 → "팔로업/크레딧 한도"로 정리.

### 2d. 불량 카드 섹션 → 범위 강조 (app/page.tsx:8 defects 배열 + :88 map 교체)
"이런 불량 겪어보셨죠?" 12종 카드 나열 = 제품을 12종짜리로 좁힘. → 다음으로 교체:
- 헤드라인: **"어떤 수지, 어떤 불량이든"**
- 서브 문구: **"범용(PP, ABS 등) / EP(PA, PC, POM 등) / Super EP(PEEK, PPA, LCP 등등) 전부 커버"**
- **불량 칩 클라우드**(작은 태그 다수, flex-wrap): 은선 · 플래시 · 웰드라인 · 싱크마크 · 버닝 · 변색 · 크랙 · 휨/변형 · 기포 · 제팅 · 박리 · GF 백화 · 흐름자국 · **+ 그 외 다수**. (큰 카드 3열 대신 칩으로 "많다=다 된다" 직관 전달)
- 안전장치 한 줄: **"목록에 없어도 사진·증상으로 AI가 판별"** (camera 아이콘, surface-sunken 배경).
- 칩 클릭 시 /diagnose 이동(선택, 기존 카드 동작 유지해도 됨).
