# OCR — 종이 표준조건표/점검표 인식 추가

## 배경 / 목적
사용자가 사출기 조작 화면 사진 대신 **종이 표준조건표(점검표)** 를 찍어 올리는 경우가 있다. 현재 `app/api/extract-settings/route.ts` 프롬프트는 "사출 성형기의 셋팅값 화면"만 전제라 종이 조건표는 인식 실패(빈 값/오독). 프롬프트 보강으로 **두 입력유형(기기 화면 / 종이 조건표) 모두** 처리하게 한다. 코드 재설계 아님 — 프롬프트 텍스트만 확장.

## 판독 정책 (진우 확정)
- **실측(주간/야간 손글씨) 우선 → 비었거나 판독 불가하면 표준조건(인쇄값)으로 폴백.**
- 주간/야간이 둘 다 있으면 **주간 우선**.
- 다단 값(1차/2차/3차/4차)은 1차를 대표값으로.

## 원칙
- 기존 스키마(반환 JSON 필드) 유지. 새 필드 추가 금지(건조온도·캐비티압력 등은 V2, 이번 범위 아님).
- 기존 화면 인식 동작 회귀 0. 종이표 케이스만 추가.
- `claude-sonnet-4-6` 모델·인증·rate limit·8MB 제한 그대로.
- 끝나면 `npx tsc --noEmit` + `npm run build` 통과, **push 금지**, 변경파일 보고.

## 수정 — app/api/extract-settings/route.ts (프롬프트 text 블록, 현 L86~134)

기존 프롬프트의 `값이 보이지 않거나 확인 불가한 항목은 빈 문자열("")로 두세요.` 문장 **바로 아래**에 다음 블록을 삽입한다:

```
입력 사진은 다음 두 가지 중 하나입니다:
(A) 사출 성형기의 조작/모니터 화면
(B) 종이에 인쇄·기입된 표준조건표 또는 점검표 (수기 기입 포함)

(B) 종이 조건표일 때 규칙:
- 조건표에는 보통 '표준조건'(인쇄된 기준값) 컬럼과 '주간/야간' 같은 실측 기입(손글씨) 컬럼이 함께 있다.
- 값 우선순위: 실측 기입값을 먼저 읽는다(주간 우선, 없으면 야간). 실측이 비어있거나 판독 불가한 항목만 표준조건(인쇄값)으로 채운다.
- 같은 항목에 1차/2차/3차/4차처럼 여러 값이 나열되면 1차 값을 대표값으로 사용한다(injSpeed1=1차, injSpeed2=2차).
- 한글 구획 → 필드 매핑:
  · 계량부 → 배압(backPressure), 계량 위치/거리(metering, 마지막 단 위치값), 냉각시간(coolTime)
  · 사출부 → 사출압력(injPressure1), 사출속도(injSpeed1/injSpeed2), 사출시간(injTime)
  · 보압부 → 보압(holdPressure), 보압시간(holdTime), 스크류 쿠션량(cushion)
  · 사출온도부 → 배럴/노즐 온도
  · 금형 → 금형온도. '상측'=고정측(moldTempFixed), '하측'=가동측(moldTempMoving). '온수기 설정온도'는 금형온도 항목이 따로 없을 때만 보조로 사용.
- 온도존 라벨은 제조사마다 N/NH/N1/N2(노즐 계열)·H1~H4·Z1~Z4 등으로 다르다. 라벨 글자에 의존하지 말고 아래 온도존 물리규칙(노즐 최고온, 호퍼/피드존 최저온)으로 매핑하라. 온도값이 5개 이상이면 내림차순 정렬해 가장 높은 값을 nozzleTemp, 그다음을 zone1~zone4로 넣는다(최저값=호퍼쪽=zone4).
```

- 기존 L117의 온도존 식별 규칙과 일관(중복돼도 무방, 강화 효과). 화면(A) 케이스는 기존 가이드 그대로 적용된다.

## 검증 — 첨부 조건표 이미지 기대 추출값 (정답 셋)

진우가 올린 표준조건표(주간 실측 기준)를 넣으면 아래에 근사해야 한다(±오독 1~2개 허용, 핵심 수치 일치 필수):

```json
{
  "nozzleTemp": "320",
  "zone1Temp": "315",
  "zone2Temp": "315",
  "zone3Temp": "305",
  "zone4Temp": "290",
  "moldTempFixed": "78.6",
  "moldTempMoving": "72.8",
  "pressureUnit": "",
  "injPressure1": "100",
  "holdPressure": "90",
  "injSpeed1": "45",
  "injSpeed2": "44",
  "holdTime": "1.5",
  "coolTime": "20",
  "injTime": "1.2",
  "metering": "56",
  "cushion": "16.2",
  "backPressure": "15",
  "screwRpm": "",
  "clampForce": ""
}
```

판정 기준:
1. 온도: nozzleTemp가 가장 높고(≈320) zone4가 가장 낮음(≈290). **노즐에 최저값(290)이 들어가면 실패**(순서 뒤집힘).
2. 금형온도 상(78.6)/하(72.8) 정상 매핑.
3. 보압 90·사출압 100·쿠션 16.2·냉각 20·배압 15 핵심 수치 일치.
4. 빈 항목(pressureUnit·screwRpm·clampForce)은 "" 유지(억지로 채우지 말 것).

## 검증 절차
1. `npx tsc --noEmit` 에러 0 + `npm run build` 통과.
2. 첨부 조건표 이미지로 추출 → 위 정답 셋과 대조(핵심 수치 일치).
3. **회귀**: 기존 기기 화면 사진 1장으로 추출 → 종전과 동일 결과(화면 인식 안 깨졌는지).
4. push 금지, 변경파일 보고.
