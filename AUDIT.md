# 프로젝트 감사 (Audit)

이 문서를 읽으면 아래 7개 영역을 순서대로 점검하고 결과를 보고해줘.
각 항목은 ✅ 통과 / ⚠️ 경고 / ❌ 실패로 표시.
마지막에 전체 점수 (통과 수 / 전체 수)와 우선 수정 목록을 정리해줘.

---

## 1. 빌드 & 실행 점검

- [ ] npm run build 성공하는가
- [ ] 빌드 경고(warning) 목록 확인
- [ ] npm run dev로 개발 서버 정상 구동하는가
- [ ] 터미널에 에러/경고 출력 없는가
- [ ] TypeScript 타입 에러 없는가 (npx tsc --noEmit)

## 2. 코드 품질

- [ ] 사용하지 않는 import 있는가 (파일별 확인)
- [ ] 사용하지 않는 변수/함수 있는가
- [ ] console.log가 남아있는가 (개발용 로그 제거 필요)
- [ ] any 타입 사용 개수 (최소화 권장)
- [ ] 하드코딩된 API 키나 비밀값이 코드에 노출되어 있는가
- [ ] .env.local이 .gitignore에 포함되어 있는가
- [ ] 모든 button에 type="button" 또는 type="submit" 명시되어 있는가
- [ ] 모든 form에 onSubmit={e => e.preventDefault()} 있는가 (불필요한 새로고침 방지)
- [ ] 중첩 객체 접근에 optional chaining (?.) 사용하고 있는가
- [ ] try-catch로 API 호출 에러 처리 되어 있는가

## 3. 파일 구조 & 아키텍처

- [ ] 폴더 구조가 CLAUDE.md에 명시된 구조와 일치하는가
- [ ] 불필요한 파일/폴더 있는가 (빈 파일, 테스트 파일 잔여물 등)
- [ ] 컴포넌트가 적절히 분리되어 있는가 (500줄 이상인 파일 확인)
- [ ] lib/ 또는 utils/ 에 공통 로직이 분리되어 있는가
- [ ] 타입 정의가 별도 파일(types.ts)에 있는가
- [ ] page.tsx 파일이 너무 크지 않은가 (300줄 이하 권장, 초과 시 컴포넌트 분리 제안)

## 4. API & 보안

- [ ] API 라우트에서 입력 검증(validation) 하고 있는가
- [ ] API 키가 서버사이드에서만 사용되는가 (클라이언트에 노출 안 되는가)
- [ ] API 응답 에러 시 사용자에게 적절한 에러 메시지 표시하는가
- [ ] rate limiting 고려가 되어 있는가 (또는 TODO로 표시)
- [ ] CORS 설정 확인
- [ ] 환경 변수 누락 시 명확한 에러 메시지 있는가

## 5. UI & UX

- [ ] 모바일 반응형 되는가 (주요 페이지별 확인)
- [ ] 로딩 상태 표시 있는가 (API 호출 시 스피너/스켈레톤)
- [ ] 에러 상태 표시 있는가 (API 실패 시 사용자 안내)
- [ ] 빈 상태(empty state) 처리 있는가 (데이터 없을 때)
- [ ] 모든 링크/버튼이 정상 작동하는가 (404 페이지 없는가)
- [ ] 폼 필수 필드 표시 및 검증 있는가
- [ ] 접근성: 이미지에 alt 속성, 버튼에 aria-label 있는가
- [ ] 페이지 간 네비게이션 일관성 (Navbar 모든 페이지에 동일)
- [ ] favicon 설정 되어 있는가

## 6. 성능

- [ ] 불필요한 리렌더링 없는가 (React DevTools 또는 코드 리뷰)
- [ ] 이미지 최적화 (next/image 사용 여부)
- [ ] 큰 라이브러리의 불필요한 전체 import 없는가 (tree-shaking 확인)
- [ ] localStorage 읽기/쓰기가 불필요하게 자주 호출되지 않는가
- [ ] API 호출이 중복 발생하지 않는가 (useEffect 의존성 확인)
- [ ] 번들 사이즈 확인 (npx next build 후 출력되는 페이지별 사이즈)

## 7. 배포 준비

- [ ] package.json의 name, version, description 설정
- [ ] README.md 있는가 (프로젝트 설명, 실행 방법)
- [ ] .gitignore에 node_modules, .next, .env.local 포함
- [ ] Vercel 배포 시 환경 변수 설정 안내 (어떤 변수가 필요한지)
- [ ] 빌드 후 정적 페이지 정상 렌더링
- [ ] 메타 태그 설정 (title, description, og:image)
- [ ] robots.txt / sitemap 설정 (SEO 필요 시)
